## СЛАЙД 2: Проблема — Теневая Экономика

**ПОТЕРИ БЮДЖЕТА**
*   70% выручки — наличные вне учета

**ОТСУТСТВИЕ КОНТРОЛЯ**
*   Нет верификации возраста
*   Отсутствие мониторинга контента (азартные игры)

**НЕТ ДАННЫХ ДЛЯ РАЗВИТИЯ**
*   Невозможно планировать индустриальную политику без статистики рынка

**«СЕРОЕ ПЯТНО» НА КАРТЕ**
*   Valve/Riot не видят узбекский рынок → нет локальных серверов

**Объем рынка:** $32-64M/год (оценка)

## СЛАЙД 4: Механизм — Гибридная Модель Учета

*Мы НЕ запрещаем наличные — мы делаем их видимыми*

**ШАГ 1: Интеграция с ПО клуба (ICAFE/runpad)**
*   Система видит реальное время работы каждого компьютера

**ШАГ 2: Сверка данных**
*   Сравнение «Время работы ПК» с «Кассовой выручкой» (онлайн + наличные)

**ШАГ 3: Сигнал о расхождениях**
*   Если ПК работал 10 часов, а выручка = 0 → флаг для налоговой проверки

**ЗАЩИТА:**
Криптографическая подпись логов (невозможно изменить данные задним числом)

## СЛАЙД 9: Риски и Митигация

**РИСК: Клубы бойкотируют систему**
*   *Вероятность:* Высокая
*   *Митигация:* Пилот с госмандатом + стимулы (турниры, трафик)

**РИСК: Манипуляция данными в ПО**
*   *Вероятность:* Средняя
*   *Митигация:* Криптоподпись логов + случайные аудиты

**РИСК: Утечка персональных данных**
*   *Вероятность:* Средняя
*   *Митигация:* Серверы в РУз + сертификация ИБ + минимальный набор данных

**РИСК: Valve/Riot отказываются**
*   *Вероятность:* Низкая
*   *Митигация:* Альтернатива — развитие локальной лиги + спонсорство

⚠️ **Ключевое условие успеха:** Государственная поддержка на этапе внедрения
